## 0. 한 줄 정의

랜덤 (팀 × 연도) 스핀으로 LoL e스포츠 역사 속 로스터를 받고, 한 명씩 드래프트해 5인 올타임 팀을 만들어 2026 시즌(Split → MSI → Worlds) 캠페인을 1초 만에 시뮬레이션하고, 결과 카드를 공유하는 무료 웹 게임.

---

## 1. 배경 & 기회

### 1.1 시장 상황 (2026-06-10 기준 검증 완료)

- 스핀-드래프트-시뮬 포맷이 바이럴 웨이브 중. 원조 38-0(EPL)은 X에서 1,000만+ 임프레션, 100만+ 방문. NBA(82-0) 클론 4개, 크리켓(11-0, 13-0) 등 버티컬이 시간 단위로 선점되는 중.
- **LoL 버티컬은 goldenroadlol.com 단 1개**, 등록 수일 이내의 인디 프로젝트.
- 직접 플레이 검증 결과 경쟁자의 약점: **선수 풀이 얇고, 선수 사진이 거의 없음** (수동 데이터 컴파일 추정). 바이럴 트랙션 증거 없음 → 카테고리 마인드셰어 무주공산.

### 1.2 우리의 비대칭 우위

1. **데이터**: Leaguepedia Cargo API로 2012~2026 전 메이저 리전 로스터 + 선수 사진 + 수상/성적 데이터를 자동 수집 가능. 경쟁자가 수동으로 못 채운 격차를 파이프라인으로 해소.
2. **한국어 + 한국 커뮤니티 접근**: LCK는 LoL 역사의 본진. 인벤/펨코/디시 네이티브 런칭은 서구 개발자가 못 하는 distribution.
3. **타이밍**: MSI 2026이 6/28~7/12 **대전(한국)** 개최. 국내 LoL 관심도 연중 최고점 구간에 런칭. 10월 Worlds(미국)가 2차 피크.

### 1.3 왜 지금인가

- 웨이브의 클론 속도상 2~3주 내 다른 진입자가 나타날 확률 높음. 약한 선점자 1명일 때 완성도 격차로 카테고리를 가져와야 함.

## 4. 핵심 게임 루프

### 4.1 플로우

```
[모드 선택] → [스핀 ×5] → [픽 ×5] → [시뮬레이션] → [결과] → [공유]
```

### 4.2 스핀 & 드래프트 규칙

1. 총 **5라운드**. 각 라운드: 슬롯머신 스핀 → **(팀 × 연도)** 조합 출현 (예: `T1 × 2016`, `G2 Esports × 2019`).
2. 해당 팀-연도의 실제 로스터가 카드로 표시됨 (사진 + 포지션 + 레이팅*).
3. 유저는 **1명** 선택. 선수는 자기 실제 포지션 슬롯(TOP/JGL/MID/ADC/SUP)에만 들어감.
4. 이미 채워진 포지션의 선수는 선택 불가 (비활성 표시).
5. **스핀 풀 사전 필터링 (v1.2)**: 스핀 후 검증→리스핀하는 거부 샘플링이 아니라, 난수 풀 자체를 "현재 빈 슬롯 포지션의 선수를 보유한 팀-연도"로 사전 필터링한 뒤 그 안에서 추첨. 무효 스핀 원천 차단 + 균등 분포 보장 + Daily 시드 재현성 확보. 빌드 타임에 팀-연도별 보유 포지션 인덱스(`spin-index.json`) 생성. **(v1.6 보강)** 여기에 런타임 필터를 추가: 이미 픽한 선수(playerId)의 다른 연도 버전만 남은 팀-연도는 풀에서 제외 — 포지션 스왑 선수로 인한 소프트락 방지. 리롤 제약 부분집합이 공집합이면 제약 해제 후 전체 유효 풀에서 추첨.
6. 리롤: **팀 리롤 1회 + 연도 리롤 1회** (런당). 업계 표준(38-0/82-0/13-0 동일)이라 학습 비용 0.
7. 동일 선수 중복 등장 가능 (다른 연도) — 이미 드래프트된 선수는 선택 불가.
8. **가중 스핀 (v1.8)**: 추첨은 균등이 아니라 팀-연도별 `weight` 비례 — 성적 기반 빌드 타임 산출 (초안: Worlds 우승 ×8 / Worlds 진출·국내 우승 ×4 / 플옵 ×2 / 그 외 ×1, §9 튜닝 대상). 근거: LoL은 축구 대비 인지 밀도가 낮아 균등 추첨 시 런당 ~3회가 무명팀 — 가중으로 체감 50~60%를 유명 팀으로 끌어올리되, **무명팀은 제거하지 않음** (재앙 런 = 공유 콘텐츠, 결정 긴장 + 올드팬 발견 재미 보존).

### 4.3 우승 마크 (WORLDS / MSI — 기존 MSI 캡슐 라벨 대체)

- 트리거: `frame === "WORLDS"`(Worlds 우승) / `msiWinner === true`(MSI 우승)
  — 기존 데이터 플래그 그대로, 스키마 변경 없음. WORLDS 프레임(§4.1)은 유지 — 마크는 추가 레이어.
- 에셋: **자체 제작 SVG 마크 2종** (`/public/mark-worlds.svg`, `/public/mark-msi.svg`).
  공식 Worlds·MSI 로고 및 소환사의 컵 실루엣 사용·모사 금지 (PRD §10-3).
  텍스트("WORLDS" / "MSI") + 단순 도형 조합의 독자 마크. currentColor로 색 수신.
- 색: accent-gold 저채도 (골드 예산 §3-9 적용). 평면·무광 — 애니메이션 금지
  (반짝이는 WORLDS 프레임 전용 원칙 유지).
- 배치: 사진 하단 페이드 존. 폭 = 카드 폭 28~34% (단일 보유 시 중앙 정렬) — lab 수렴 후 확정.
- **겹침 규칙 (둘 다 보유 시)**: WORLDS 좌 → MSI 우 고정 순서, MSI가 WORLDS 우측 하단에
  12~15% 오버랩, z-order·오프셋 고정 파라미터로 전 카드 동일 (결정론 — 카드별 변형 금지).
- 왕관과 시각 무게 경쟁 금지. lab 판정 기준: ① slot 사이즈에서 마크 식별 가능
  ② 오버랩 상태가 소형에서 뭉개지지 않음.

### 4.4 로스터 데이터 규칙 (엣지 케이스)

- 한 팀-연도에 같은 포지션 선수가 2+명(시즌 중 교체)일 경우: 해당 시즌 **출전 경기 수 상위자만 노출**, 단 출전 5경기 이상이면 모두 노출.
- 2011~2012 초기 시대는 포지션 구분이 흐림 → 데이터 검수 후 포함 여부 결정 (기본: **2013년부터 시작**, §13 오픈 퀘스천).
- 팀 리브랜딩(SKT→T1, 삼성→Gen.G)은 연도 시점의 팀명으로 표기.

## 5. 시뮬레이션 스펙

### 5.1 캠페인 구조 — "2026 글로벌 캘린더" 고정

유저 팀이 2026년에 소환됐다는 컨셉. 진행 순서:

```
국내 Split 1 (정규 + 플옵) → MSI → 국내 Split 2 → Worlds (스위스 → 녹아웃)
```

- 상대 팀 풀 (v1.5 간소화): 리그 매칭 없는 **글로벌 고정 봇 풀 2개** — 정규 시즌용 9팀 + 국제전용 12팀 내외, 각자 고정 레이팅. 유저 팀이 5개 시대·리그 혼성이라 "자국 리그"가 정의 불가하고, 현실 리그별 팀 수 차이(LPL 16+팀 등) 대응 로직도 불필요해짐.
  - 각 봇 팀 필드: `name`(내부 식별자), `label`(결과 화면 표시용 상징명 — 예: `"16 SKT"`, `"21 DK"`), `league`(플레이버 문자열), `rating`(60~99). `label`은 D3 호빈이 실값 입력 시 함께 채움.
- 최종 결과 등급: `Split 우승 → MSI 우승 → Worlds 우승` 트로피 개수 + 탈락 지점. 전부 우승 = **GRAND SLAM** (최고 등급, 희귀하게 설계).

### 5.2 매치 엔진

```
팀 전력 teamPower = Σ (선수 레이팅 × 포지션 가중치)
포지션 가중치 v0: MID 1.10 / JGL 1.10 / ADC 1.00 / TOP 0.95 / SUP 0.85 (합계 5.0 정규화)
팀 OVR = teamPower / 5        // 60~99 스케일 — 승률 비교는 반드시 이 정규화 값으로 (합산값 직접 비교 금지)

게임 승률 P = 1 / (1 + 10^((상대 팀 OVR − 우리 팀 OVR) / S))   // 양변 동일 60~99 스케일, S는 튜닝 상수
시리즈: 정규시즌 Bo1·Bo3, 토너먼트 Bo5 — 게임 단위로 난수 시뮬
```

- **연산은 1초 미만** (클라이언트 사이드 순수 계산). 결과 공개는 단계별 리빌 애니메이션 (스킵 버튼 필수).
- 시너지/시대 보정: v1에서는 **넣지 않음**. 시대 섞기가 재미의 본체이므로 패널티 금지.

### 5.3 정합성 기준

- 평균적 픽 → Worlds 8강~4강. 최상위 픽(전부 95+) → Grand Slam 확률 약 15~25%. 약한 픽 → Split 플옵 탈락 가능.
- 튜닝 방법: 몬테카를로 10,000회 × 픽 정책 2종 — ① 그리디(노출 로스터에서 최고 OVR 선택, 실제 유저 근사 = **목표 판정 기준**) ② 균등 랜덤(베이스라인). (스크립트 1개)

## 6. 레이팅 & 선수 카드 시스템 (신뢰성·비주얼의 코어)

> **결정 (v1.1)**: FIFA식 세부 스탯(6각형)은 미채택. **OVR 단일 수치 + FIFA풍 카드 비주얼**만 MVP에 포함. 세부 지표 보강(Oracle's Elixir)은 P2.

### 6.1 산출 공식 v0 (선수-시즌 단위, 자동 산출)

```
base = 60
+ 국내 플옵 최종 순위: 우승 +10 / 준우승 +6 / 4강 +3 / 5~6위 +3 / 7위 이하 +1   // TournamentResults(팀 단위) → 로스터 전개, 조인 1개
+ MSI 최종 순위: 우승 +8 / 결승 +5 / 4강 +3
+ Worlds 최종 순위: 우승 +15 / 결승 +10 / 4강 +7 / 8강 +4 / 진출 +2
+ 수상 보너스 (awards.csv): 시즌 MVP +6 / 파이널 MVP +4 / Worlds MVP +8
                            All-Pro 1st +5 / 2nd +3 / 3rd +1
+ EDITORIAL ±1~5 (awards.csv value 컬럼 — 앵커 보정 전용, 최대 100건)
cap 99 / floor 60
```

※ 가중치는 v1.7 초안 — **D0에 §6.2 앵커 10개를 수기 계산(30분)해 1회 확정 후 파이프라인 가동** (생략 시 D3 앵커 세션이 공식 재설계로 번짐).

**v1.7 분포 보강:** 기존 공식은 플옵 4강 미만 + 무수상 선수의 가점이 0이라 **대다수 선수-시즌이 OVR 60에 뭉치는 분포 결함**이 있었음 (카드 풀 사망 + 몬테카를로 목표 구조적 미달). 하위 순위 가점(동일 순위 테이블 사용, 조인 추가 0)으로 바닥을 분산. 초기 시대(2013~2015) 수상 제도 부재로 인한 구조적 저평가는 EDITORIAL로 앵커 한정 보정 — §6.3에 "성적 기반 + 편집 판단"을 이미 공표하므로 정직한 방식. base 70 리베이스는 비채택 (상위권 99 압착으로 엘리트 변별 붕괴).

**v1.2 단순화 결정:**

- **정규시즌 승률 항목 제거** — ScoreboardPlayers 대량 조인 비용 대비 변별 기여가 낮음. 팀의 "대회 최종 순위"만 사용해 Cargo 조인을 1개로 축소.
- **수상 데이터는 Cargo 다중 조인이 아니라 별도 정적 `awards.csv`로 분리** — 전체 수량이 ~1,000건 내외(시즌 MVP는 리그당 연 2명 수준)라 위키 페이지 반자동 수집이 조인 파싱보다 빠르고 검수 가능.
- **수상 레이어는 제거 불가 (중요)** — 팀 순위만 쓰면 같은 우승 로스터 5인이 전원 동일 OVR이 되어(Faker 2016 = 서브 멤버) ICON 티어와 카드 변별력이 붕괴함. 수상 보너스가 유일한 개인 변별 장치.

### 6.2 앵커 튜닝 (D0 수기 계산 → 출시 전 1회 검증)

커뮤니티가 납득해야 할 기준 선수-시즌 10개. **목표값은 시대별 제도 차이를 반영해 현실화 (v1.7)** — MSI(2015~)·공식 수상 제도 부재 시대는 상한이 구조적으로 낮음:

- Faker 2016 → 97~99 (전관왕 + 수상 싹쓸이 시대)
- Faker 2013 → 93~95 (MSI·공식 Worlds MVP 제도 부재 시대 — **전제: 2013 국내 수상(Champions 결승 MVP 등) 존재 여부를 awards 큐레이션에서 조사.** 수상 1건 + EDITORIAL로 도달. 미발견 시 앵커 90~93 완화 — EDITORIAL ±8 확대는 신뢰 서사 훼손으로 비채택. D0 세션 결정)
- Canyon 2020, Chovy 2024 → 92~96
- Ruler 2017 → 88~92
- 평범한 중위권 팀 주전 → 60대 중반 (5~6위 가점 기준 63이 하단 경계 — D0 수기 세션에서 가점 수치와 함께 확정)
- 절차: **D0에 스프레드시트 수기 계산 30분 → 가중치 1회 확정 → 파이프라인 가동.** 이후 어긋나면 가중치 조정 → 재산출 (파이프라인이므로 비용 낮음)

### 6.3 레이팅 논쟁 = 콘텐츠

- 푸터에 "레이팅은 성적 기반 자동 산출 + 편집 판단" 명시. 산출 기준 페이지(About) 공개 → 논쟁을 트래픽으로 전환.
- 명백한 오류 제보 채널 (구글 폼 1개) + 빠른 패치를 운영 루틴으로.

### 6.4 선수 카드 스펙 (P0)

게임의 모든 화면(드래프트 픽, 결과, OG 공유 이미지)에서 선수는 **단일 카드 컴포넌트**로 표현. 스핀 → 카드 공개 연출은 팩 개봉 감성으로 설계 (이 게임의 도파민 코어).

**카드 구성 요소:**

| 요소 | 내용 |
|---|---|
| OVR | 좌상단 대형 숫자 (카드의 시각적 중심) |
| 포지션 | OVR 아래 (TOP/JGL/MID/ADC/SUP) |
| 사진 | 중앙 — **해당 시즌 사진 우선** (2016 Faker는 2016 사진). 없으면 이니셜 + 팀 컬러 아바타 폴백 |
| 이름 | EN/KR 토글 연동 (Faker / 페이커) |
| 팀 × 연도 | 하단 (예: `SKT T1 · 2016`) |

**장식 레이어 시스템 (v1.3 — OVR 구간별 티어 4종 폐기, 업적 직결 레이어로 교체):**

| 레이어 | 트리거 | 비주얼 | 추정 희소성 |
|---|---|---|---|
| 프레임 | **Worlds 우승 시즌** | 유일한 "반짝이" 처리 — 그라디언트 보더 + 시머 스윕. 그 외 전부 일반 다크 프레임 1종 | ~2% |
| 왕관 | **FINALS_MVP 또는 WORLDS_MVP 수상 시즌** (SEASON_MVP 제외, 중복 비누적 — 왕관은 항상 1개) | 의도적으로 조악한 **그림판풍 손그림 왕관 PNG 1종**(`/public/crown.png`)을 사진 좌상단에 삐딱하게(-12°) 오버레이. 밈 장치이므로 다듬지 않음 | ~4% |
| 라벨 | **MSI 우승 시즌** | 사진 하단 소형 캡슐 `MSI WINNER` (영문 고정) | ~1.5% |
| 배지 | 리그 우승 / All-Pro 1st | 우상단 아이콘 최대 2개 (Worlds·MSI·MVP는 위 레이어가 담당하므로 배지에서 제외) | — |

- 레이어는 전부 **중첩 가능** — Faker 2016 = 시머 프레임 + 왕관 + MSI 라벨 + 배지 풀셋이 자연스럽게 최고 연출이 됨 (별도 등급 불필요).
- 일반 카드(프레임 없음)가 화면의 **95%+** — 일반 프레임 1종의 완성도가 게임 전체 인상을 결정. 왕관의 조악함은 "일관된 조악함"이어야 함: 에셋 1개를 전 카드 재사용, 변형 금지.

**디자인 제약:** "FIFA풍 등급 카드" 컨셉은 차용하되, EA(EA SPORTS FC) 카드의 레이아웃·형태를 모방하지 않는 독자 디자인 필수 (§10-6). 카드 1종 컴포넌트를 드래프트/결과 화면에서 재사용. OG 이미지는 렌더링 제약상 단계화 — §8 참조 (P0: 타이포 중심 / P1: 카드 그리드).

## 7. 데이터 파이프라인

### 7.1 소스: Leaguepedia (lol.fandom.com) Cargo API

- 확인된 테이블/필드: `Players (ID, Image, Name, Team, Role)`, `Teams (Name, Region)`, `Tenures`, `ScoreboardPlayers` (경기 단위 출전 기록), `Tournaments`.
- 선수 사진 URL: `Special:Filepath/{Image}` 패턴.
- 수상(MVP/All-Pro) 테이블명은 **D0 스키마 탐색에서 확정** (없으면 위키 페이지 파싱으로 대체).
- 쿼리 제약: 익명 cargoquery 1회 500행 → 분할 쿼리 + 스로틀 필수. 래퍼(`poro` 등) **미사용** — `scripts/lib/cargo.ts` 자체 구현 (가이드 §4.1).

### 7.2 수집 범위 (P0)

| 항목 | 범위 |
|---|---|
| 리전 | LCK, LPL, LEC(舊 EU LCS), LCS(舊 NA LCS → 2025~ LTA North) |
| 기간 | 2013 ~ 2025 (**2026 진행 시즌 제외** — 성적 미완결로 현역 스타 OVR이 왜곡됨. Worlds 업데이트(10월) 때 "2026 시즌 카드 추가"를 재푸시 콘텐츠로 활용. 2012 이전 제외) |
| 국제전 | Worlds, MSI (First Stand 제외 — Q4 확정) |
| 추정 규모 | 선수-시즌 약 4,000~7,000건 (파이프라인 실행 후 확정) |

### 7.3 산출물

```
/data
  players.json        // 선수-시즌: id, 이름(EN/KR), 팀, 연도, 포지션, 레이팅, 사진URL(R2), 주요성적 태그
                    // ※ nameKo는 점진 보강 필드 — 출시 게이트 아님. null이면 UI가 nameEn 폴백 표시.
  teams.json          // 팀-연도: 팀명, 리전, 로스터 참조
  opponents-2026.json // 시뮬 상대 팀 풀 (고정 티어표)
  spin-index.json     // 팀-연도별 보유 포지션 인덱스 (스핀 풀 사전 필터링용, §4.2-5)
/pipeline-input
  awards.csv          // 수상 데이터 (위키 반자동 수집·검수) — 레이팅 산출 입력
```

- **빌드 타임 1회 수집 → 정적 JSON 번들.** 런타임 API 호출 없음 (속도·안정성·API 부하 0).
- **사진 셀프 호스팅 확정 (v1.2)**: Fandom 핫링크는 Referer 검증·CDN 정책 변경 리스크로 **금지 전제**. 빌드 파이프라인에서 다운로드 → 256px webp 리사이즈 → **Cloudflare R2** 업로드(이그레스 무료, 추정 용량 ~250MB로 무료 티어 내), players.json에는 R2 URL만 기록. **사진 OFF 킬스위치**(환경변수 플래그)는 P0 유지.
- **시즌별(era-correct) 사진 우선 (v1.5)**: Leaguepedia의 시대별 선수 사진 아카이브에서 해당 연도·팀 사진을 매칭 (2013 페이커 ≠ 2026 페이커). 폴백 체인: 같은 팀·연도 → 같은 팀·최근접 연도 → 현재 프로필 → 아바타. 올드팬 페르소나(§3)의 핵심 트리거이자 경쟁자 대비 차별점.
- 사진 없는 선수: 이니셜 + 팀 컬러 아바타 폴백 (자동 생성).

## 10. 법적 · 정책 가드레일 (위반 시 런칭 불가)

1. **비상업 유지**: 광고·유료화 금지. 허용 한도는 도네이션 링크까지 (경쟁자와 동일 모델). 이 제약은 Riot 팬 콘텐츠 정책 준수 + 선수 퍼블리시티권 리스크 최소화의 전제 조건.
2. 풋터 고정 문구: *"○○ is a fan-made project. Not affiliated with or endorsed by Riot Games. League of Legends is a trademark of Riot Games, Inc."* (KR 병기)
3. Riot 로고·게임 아트 사용 금지. 사용하는 것은 공개된 사실 데이터(이름·성적)와 Leaguepedia 경유 선수 사진뿐.
4. **사진 킬스위치**: 권리자 요청 시 환경변수 1개로 전체 사진 → 아바타 전환 + 24시간 내 대응 원칙.
5. 선수 비하 표현 금지 — 낮은 레이팅도 표현은 중립("성적 기반 산출")으로.
6. **카드 디자인 독자성**: EA SPORTS FC(舊 FIFA) 카드의 레이아웃·트레이드 드레스를 모방하지 않음. "등급별 카드" 컨셉만 차용하고 형태·구도는 자체 디자인.
